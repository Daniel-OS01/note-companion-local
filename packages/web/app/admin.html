Access Control

Manage the password wall and user management settings.

🛡️ HTTP BasicAuth (browser password wall)
Protects the web UI via Traefik. The Obsidian plugin is unaffected because /api/* routes bypass this check entirely.
ℹ️
The password is stored as a one-way hash. To change it, generate a new hash and update TRAEFIK_BASIC_AUTH_USERS in Dokploy. No restart needed — Traefik picks up label changes on redeploy.

⚠️ This is used only in your browser to compute the MD5 hash. The password itself is never sent anywhere or saved.

Remember: every $ in the hash must be written as $$ in Dokploy's env panel.

👤 User Management (Clerk)
Controls whether users must sign in via Clerk to use the dashboard.

Must be exactly the string true to enable. Any other value, including the string false, disables it.

How to Apply Changes

Step-by-step guide for deploying configuration updates via Dokploy.

📋 Workflow for any configuration change
Follow these steps every time you update a setting in this panel.
  1. Use the "Copy to clipboard" button on whichever section you modified. This copies the env var lines in KEY=VALUE format.
  2. Open Dokploy → select your application → click Environment Variables.
  3. Find the variable(s) you want to change and update their values. Or add new lines if they don't exist yet.
  4. Click Save in Dokploy's env panel.
  5. Redeploy the container. In Dokploy, click the Deploy button. Dokploy will pull the latest image and start a new container with the updated environment.
  6. Verify by going to the System Status section of this panel and clicking Load Current Values. Confirm the new values appear.
⚠️
Model changes always need a restart. The app reads MODEL_PROVIDER and MODEL_NAME at startup in lib/models.ts. They are cached in module-level constants and do not re-read from the environment until the Node.js process restarts.
🔌 Why the Obsidian plugin still works
Understanding the two-router Traefik setup.

The Traefik configuration in docker-compose.yml defines two routers for the same container. The API router matches any URL starting with /api and has priority 20 — it has no BasicAuth middleware attached. The UI router matches everything else at priority 10 and applies the BasicAuth middleware. Because Traefik evaluates routers in priority order, plugin requests to /api/organize, /api/health, etc. always hit the API router and are never challenged for a browser password. Their own authentication is handled by the existing handleAuthorizationV2 Bearer-token system.

🔄 Generating a new BasicAuth password hash
Use the Access Control section, or run this command locally.
# Run this in your terminal — outputs:  admin:$apr1$xyz...$abc...
htpasswd -nb admin yourNewPassword

# In Dokploy, set:
TRAEFIK_BASIC_AUTH_USERS=admin:$$apr1$$xyz...$$abc...
# (double every $ sign)
